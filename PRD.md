# PRD: 온라인 책 판매 쇼핑몰

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | BookShop (가칭) |
| 목표 | 일반 소비자(B2C) 대상 온라인 책 판매 쇼핑몰 |
| 기술 스택 | Next.js (App Router), Supabase (DB + Auth + Storage), 토스페이먼츠 (결제) |
| 타겟 사용자 | 책을 구매하려는 일반 소비자 + 운영 관리자 |

## 2. 사용자 역할

| 역할 | 설명 |
|------|------|
| **비회원** | 책 검색/조회 가능, 구매 시 로그인 필요 |
| **회원** | 책 구매, 리뷰 작성, 주문 내역 확인, 위시리스트 |
| **관리자** | 상품/주문/회원 관리, 대시보드 |

## 3. 핵심 기능

### 3.1 인증 (Supabase Auth)

- 이메일/비밀번호 회원가입 및 로그인
- 소셜 로그인 (Google, Kakao)
- 비밀번호 재설정 (이메일)
- 프로필 관리 (이름, 연락처, 배송지 주소)

### 3.2 책 목록 및 검색

- **카테고리 브라우징**: 대분류(소설, 비소설, 자기계발 등) → 소분류
- **검색**: 제목, 저자, ISBN으로 검색 (Supabase Full-text Search)
  - 실시간 자동완성(debounce 300ms)으로 검색어 입력 즉시 결과 미리보기 → 클릭 한 번으로 상세 이동
- **정렬**: 최신순, 인기순, 가격순, 평점순
- **필터**: 카테고리, 가격대, 평점, 출판일 — URL 쿼리 파라미터 연동으로 별도 "적용" 버튼 없이 즉시 반영
- **페이지네이션**: 무한스크롤 (스크롤만으로 다음 목록 로드, 페이지 이동 클릭 불필요)

### 3.3 책 상세 페이지

- 책 정보: 제목, 저자, 출판사, 출판일, ISBN, 페이지 수, 가격
- 책 표지 이미지 (Supabase Storage)
- 책 소개/설명
- 리뷰 및 평점 표시
- 관련 도서 추천 (같은 카테고리/저자)
- 장바구니 담기 / **바로 구매** 버튼 (바로 구매 시 장바구니를 거치지 않고 즉시 주문서로 이동)
- 수량 선택 + 장바구니 담기를 한 줄에 배치하여 원클릭 완료

### 3.4 장바구니

- 책 추가/삭제/수량 변경
- 소계 및 총 금액 계산
- 비회원 → 로그인 시 장바구니 유지 (localStorage → DB 동기화)
- 장바구니 추가 시 페이지 이동 없이 토스트 알림으로 확인 (현재 페이지 유지)
- 헤더의 장바구니 아이콘 호버/클릭 시 미니 장바구니 드롭다운 표시 → 바로 결제 가능

### 3.5 주문 및 결제

- **주문 프로세스**: 장바구니 → 주문서 작성/결제 → 주문 완료 (주문서와 결제를 **단일 페이지**에서 처리)
- **배송 정보 입력**: 기본 배송지 자동 채움 — 변경이 필요한 경우에만 수정 (클릭 0회로 배송지 입력 완료)
  - 수령인, 연락처, 주소 (다음 우편번호 API)
  - 최근 배송지 목록에서 원클릭 선택
- **결제 연동**: 토스페이먼츠
  - 카드 결제
  - 계좌이체
  - 간편결제 (토스페이, 카카오페이, 네이버페이)
  - 마지막 결제 수단 기억 → 다음 결제 시 자동 선택
- **주문 상태**: 결제완료 → 배송준비 → 배송중 → 배송완료
- **주문 취소/환불**: 배송 전 취소 가능, 환불 처리
- **재주문**: 주문 내역에서 "다시 주문하기" 원클릭 → 동일 상품 장바구니 담기 + 주문서 이동

### 3.6 주문 내역

- 주문 목록 (날짜, 상태, 금액)
- 주문 상세 보기
- 배송 추적 (외부 링크)
- 주문 취소 요청

### 3.7 리뷰 및 평점

- 구매 확정 후 리뷰 작성 가능
- 별점 (1~5점) + 텍스트 리뷰 — 별점만으로도 리뷰 등록 가능 (텍스트 선택 사항, 최소 클릭으로 평가 완료)
- 리뷰 수정/삭제
- 리뷰 도움이 됐어요 (좋아요)
- 책별 평균 평점 자동 계산

### 3.8 관리자 대시보드

- **상품 관리**: 책 CRUD (제목, 가격, 재고, 이미지, 설명 등)
- **주문 관리**: 주문 목록, 상태 변경, 환불 처리
- **회원 관리**: 회원 목록, 상태 변경 (활성/정지)
- **리뷰 관리**: 부적절한 리뷰 삭제
- **대시보드 홈**: 매출 요약, 최근 주문, 신규 가입 등 통계

## 6. UX/UI 디자인 (인디 스타일)

### 6.1 디자인 컨셉

독립 서점의 따뜻하고 개성 있는 분위기를 디지털로 구현한다. 대형 쇼핑몰의 차갑고 기능 중심적인 UI가 아닌, 소규모 독립 서점에서 책을 고르는 듯한 감성과 경험을 제공한다.

### 6.2 컬러 팔레트

| 용도 | 색상 | 설명 |
|------|------|------|
| 배경 | `#FDF6EC` (크림) | 오래된 종이/양피지 느낌의 따뜻한 배경 |
| 주 텍스트 | `#2C2C2C` (차콜) | 잉크 느낌의 진한 텍스트 |
| 보조 텍스트 | `#6B5B4E` (웜 브라운) | 부드러운 갈색 계열 보조 텍스트 |
| 강조색 | `#C45D3E` (테라코타) | CTA 버튼, 링크, 가격 등 핵심 강조 |
| 보조 강조 | `#5B7F5E` (올리브 그린) | 태그, 뱃지, 할인 표시 |
| 카드/섹션 배경 | `#FAF0E4` (밝은 베이지) | 카드 및 구분 영역 배경 |
| 보더 | `#E0D5C7` (샌드) | 구분선, 카드 테두리 |

### 6.3 타이포그래피

- **제목 (h1~h3)**: 세리프 계열 (Noto Serif KR 또는 Playfair Display) — 클래식한 서점 느낌
- **본문/UI 텍스트**: 산세리프 계열 (Pretendard 또는 Noto Sans KR) — 가독성 확보
- **특수 장식**: 필기체/손글씨 느낌 (Gowun Batang) — 추천 코멘트, 인용문 등 포인트 요소

### 6.4 레이아웃 및 구성 원칙

- **여백 넉넉하게**: 요소 간 충분한 여백으로 시각적 여유와 고급스러움 표현
- **비대칭 그리드**: 정형화된 그리드 대신 약간의 비대칭 배치로 수제 느낌 부여
- **둥근 모서리 최소화**: 각진 모서리 또는 살짝 둥근 정도(`border-radius: 2~4px`)로 빈티지 감성 유지
- **카드 디자인**: 그림자 대신 얇은 보더 또는 미세한 텍스처로 깊이 표현
- **최대 너비 제한**: 콘텐츠 영역 `max-width: 1080px` — 넓게 퍼지지 않는 집중된 읽기 경험

### 6.5 클릭 최소화 설계 원칙

사용자가 원하는 결과에 도달하기까지의 클릭 수를 최소화한다. 모든 주요 플로우는 아래 클릭 예산을 기준으로 설계한다.

#### 주요 플로우별 클릭 예산

| 플로우 | 최대 클릭 수 | 설명 |
|--------|-------------|------|
| 검색 → 구매 완료 | 4클릭 | 검색 결과 클릭 → 바로 구매 → 결제 → 완료 |
| 목록 → 장바구니 담기 | 1클릭 | 목록 카드에서 바로 장바구니 추가 (상세 진입 불필요) |
| 장바구니 → 결제 완료 | 2클릭 | 결제하기 → 결제 승인 |
| 재주문 | 2클릭 | 다시 주문하기 → 결제 승인 |
| 리뷰 작성 (별점만) | 1클릭 | 별점 클릭만으로 등록 |

#### 클릭 절감 패턴

- **인라인 액션**: 목록 카드에 장바구니 추가 버튼을 직접 배치하여 상세 페이지 진입 없이 담기 가능
- **미니 장바구니**: 헤더 장바구니 아이콘 클릭 시 드롭다운으로 담긴 상품 확인 + 바로 결제 진행
- **자동 채움**: 기본 배송지, 최근 결제 수단 자동 선택으로 입력/선택 클릭 제거
- **바로 구매**: 상세 페이지에서 장바구니를 거치지 않고 즉시 주문서로 이동
- **페이지 이동 최소화**: 장바구니 담기 시 토스트 알림만 표시하고 현재 페이지 유지
- **모달 활용**: 주소 변경, 리뷰 작성 등은 페이지 이동 대신 모달/슬라이드 패널 사용
- **무한스크롤**: 페이지네이션 클릭 없이 스크롤만으로 다음 목록 로드
- **실시간 필터**: "적용" 버튼 없이 필터 선택 즉시 결과 반영
- **퀵뷰**: 목록에서 책 표지 호버 시 간략 정보 팝오버 표시 (가격, 평점, 한줄 소개)

### 6.6 핵심 페이지별 UX

#### 홈페이지 (/)
- 히어로 영역: 손글씨 스타일의 추천 문구 + 큐레이션 도서 1~3권 (대형 배너가 아닌 '서점 주인의 추천' 느낌)
- "이번 주의 책", "새로 들어온 책", "독자들의 선택" 등 감성적인 섹션 타이틀
- 책 표지 중심의 시각적 배치, 가로 스크롤 캐러셀
- 각 큐레이션 카드에 장바구니 추가 버튼 직접 배치 → 홈에서 바로 담기 가능

#### 책 목록 (/books)
- 그리드 뷰(기본) + 리스트 뷰 전환 가능
- 책 카드: 표지 이미지 + 제목 + 저자 + 가격 + **장바구니 추가 버튼** (상세 진입 없이 바로 담기)
- 호버 시 부드러운 확대 효과 + 퀵뷰 팝오버 (평점, 한줄 소개, 바로 구매 버튼)
- 필터/정렬은 드롭다운으로 깔끔하게, 선택 즉시 반영 (별도 적용 버튼 없음)

#### 책 상세 (/books/[id])
- 좌측: 책 표지 (큰 이미지), 우측: 책 정보
- 책 소개 영역은 실제 책 속 한 구절 인용 스타일로 표현
- 리뷰 영역: 손글씨/메모 느낌의 카드 형태, 별점 클릭만으로 간편 리뷰 등록
- 수량 선택 + "장바구니에 담기" + "바로 구매" 를 한 줄에 나란히 배치
- "장바구니에 담기" 클릭 시 페이지 이동 없이 토스트로 확인

#### 장바구니 (/cart)
- 심플한 리스트 형태, 수량 조절은 +/- 버튼 (변경 즉시 금액 반영, 별도 업데이트 버튼 없음)
- 하단 고정 주문 요약 바 (모바일) — "결제하기" 버튼 항상 노출
- 개별 삭제는 스와이프(모바일) 또는 X 버튼 원클릭

#### 주문서/결제 (/checkout)
- 배송지 + 결제를 **단일 페이지**에서 처리 (단계 분리 없음)
- 기본 배송지 자동 채움, 변경 시 모달로 주소 선택
- 마지막 결제 수단 자동 선택
- "결제하기" 한 번 클릭으로 결제 완료

#### 미니 장바구니 (헤더 드롭다운)
- 헤더 장바구니 아이콘 클릭 시 드롭다운으로 담긴 상품 미리보기
- 수량 변경, 삭제, "전체 결제하기" 가능 → 장바구니 페이지 방문 불필요

### 6.7 인터랙션 및 애니메이션

- **전환 효과**: 페이지/요소 전환 시 부드러운 페이드(fade) 적용 (`200~300ms`)
- **호버 효과**: 책 표지 미세 확대 (`scale 1.02~1.05`), 버튼 색상 부드러운 전환
- **로딩**: 스켈레톤 UI 대신 심플한 펄스 애니메이션 또는 책 아이콘 로딩 인디케이터
- **토스트 알림**: 장바구니 추가 등 액션 피드백은 화면 하단에 부드럽게 등장/퇴장
- **과도한 모션 지양**: 인디 감성에 맞게 절제된 애니메이션, 화려함보다 자연스러움

### 6.8 아이콘 및 그래픽 요소

- **아이콘**: 라인 아이콘 스타일 (Lucide Icons 또는 Phosphor Icons) — 손그림 느낌의 가벼운 스타일
- **장식 요소**: 페이지 구분선에 작은 책/식물 일러스트, 섹션 사이 미세한 텍스처 패턴
- **빈 상태(Empty State)**: "장바구니가 비어있어요" 등에 감성적인 일러스트와 따뜻한 안내 문구

### 6.9 반응형 디자인

- **모바일 (< 640px)**: 단일 컬럼, 하단 네비게이션 바, 터치 친화적 버튼 크기 (최소 44px)
- **태블릿 (640~1024px)**: 2컬럼 그리드, 사이드 메뉴
- **데스크톱 (> 1024px)**: 3~4컬럼 그리드, 상단 네비게이션

## 9. 비기능 요구사항

- **반응형 디자인**: 모바일, 태블릿, 데스크톱 지원
- **SEO**: 책 상세 페이지 SSR/SSG, 메타 태그, Open Graph
- **성능**: 이미지 최적화 (Next.js Image), 페이지 로딩 3초 이내
- **보안**: RLS (Row Level Security) 적용, XSS/CSRF 방지, 결제 검증
- **개인정보 마스킹**: 화면에 개인정보를 표시할 때 반드시 마스킹 처리 적용
  - **이메일**: 앞 3자리 노출 후 `@` 앞까지 마스킹 (예: `tes***@gmail.com`)
  - **휴대폰 번호**: 가운데 4자리 마스킹 (예: `010-****-5678`)
  - **이름**: 첫 글자와 마지막 글자만 노출, 가운데 마스킹 (예: `홍*동`, `김*`)
  - **주소**: 상세 주소 전체 마스킹, 시/구까지만 노출 (예: `서울시 강남구 ***`)
  - **결제 카드번호**: 앞 4자리 + 뒤 4자리만 노출 (예: `1234-****-****-5678`)
  - **주문번호**: 마스킹 없이 전체 노출 (개인정보 아님)
  - 마스킹은 서버 측(API 응답)에서 처리하며, 프론트엔드에 원본 데이터를 전달하지 않음
  - 관리자 페이지에서도 동일하게 마스킹 적용 (단, "상세보기" 클릭 시 권한 확인 후 원본 노출 가능)
- **접근성**: 기본 웹 접근성 준수 (시맨틱 HTML, alt 텍스트)
